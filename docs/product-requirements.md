# ytDownloader 可视化桌面端需求文档

版本：v0.1  
日期：2026-03-16  
状态：初稿

## 1. 项目背景

`yt-dlp` 是一个功能非常强的命令行音视频下载工具，支持数千个站点，并且具备格式选择、字幕下载、封面写入、Cookie 登录、播放列表下载、后处理等能力。  
它的主要问题不是能力不足，而是普通用户的使用门槛高：

- 需要理解命令行参数 
- 需要自己处理 `ffmpeg`、Cookie、输出目录、格式选择
- 出错时日志不直观
- 批量任务、失败重试、历史记录等流程不够友好

因此，本项目目标不是重写下载内核，而是基于 `yt-dlp` 做一个“真正可用”的桌面可视化外壳，让普通用户能稳定完成下载任务，让进阶用户也能逐步接触高级能力。

## 2. 产品定位

这是一个基于 `yt-dlp` 的桌面 GUI 下载器，面向本地下载场景，核心定位如下：

- 对新手：复制链接即可下载
- 对常用用户：可以看见清晰的格式、字幕、封面、保存路径与任务进度
- 对进阶用户：保留 `yt-dlp` 的关键能力，但通过图形化配置逐步暴露，而不是一次性堆满所有选项

结论：产品应优先做桌面端，而不是 Web 端。

原因：

- 本地文件保存是核心需求
- 需要调用本机 `yt-dlp`、`ffmpeg`
- Cookie 读取、浏览器配置、系统权限更适合桌面应用
- 下载队列、历史记录、日志诊断也更适合桌面端

## 3. 产品目标

## 3.1 业务目标

- 将 `yt-dlp` 的常用能力可视化，显著降低使用门槛
- 让用户无需记忆参数即可完成 80% 以上常见下载任务
- 建立一套稳定的下载任务中心、错误提示和基础设置体系
- 为后续高级能力扩展预留结构，不在首版把 CLI 全量参数搬进 UI

## 3.2 用户目标

- 复制一个链接后，快速拿到可下载内容
- 能清楚知道将下载什么、保存到哪里、下载进度如何
- 在视频、音频、字幕、封面、播放列表之间做直观选择
- 在登录/需要 Cookie 的站点中完成可控下载
- 出错时能知道下一步该怎么处理

## 3.3 成功指标

- 首次安装后 5 分钟内完成首个下载
- 单链接下载成功率达到可用水平
- 常见失败原因可被界面识别并给出明确提示
- 用户无需查看命令文档即可完成以下任务：
  - 下载单个视频
  - 提取音频
  - 选择清晰度/格式
  - 下载字幕
  - 下载播放列表
  - 设置保存目录

## 4. 目标用户

## 4.1 主要用户

- 希望下载 YouTube 或其他视频站内容的普通用户
- 不熟悉命令行，但能理解“画质/格式/字幕/保存目录”等概念的用户
- 需要批量下载课程、播客、合集、播放列表的内容整理型用户

## 4.2 次要用户

- 已经会用 `yt-dlp`，但希望用 GUI 管理任务与配置的进阶用户
- 需要更方便处理 Cookie、字幕、封面、转码的内容采集用户

## 5. 产品原则

- 简单优先：默认界面只展示高频功能
- 渐进增强：高级功能放入“高级设置”或“更多选项”
- 透明可信：任务进度、输出路径、实际下载格式、错误原因必须可见
- 稳定优先：优先支持稳定下载，而不是追求参数覆盖率
- 可恢复：失败任务可重试，历史任务可追溯
- 不误导：不承诺 `yt-dlp` 本身不具备或不稳定的能力

## 6. 范围定义

## 6.1 MVP 范围

MVP 只覆盖“高频且稳定”的下载能力：

- 单个视频/音频下载
- 播放列表下载
- 批量链接下载
- 下载前解析视频信息
- 格式/清晰度选择
- 音频提取与转码
- 字幕下载
- 封面下载与基础元数据写入
- 任务队列、进度、取消、重试
- 保存目录设置
- Cookie 文件或浏览器 Cookie 导入
- 基础日志与错误提示
- `yt-dlp` / `ffmpeg` 环境检测
- YouTube 场景下的 JavaScript runtime 环境检测

## 6.2 首版不做

- 账号系统、云端同步
- 在线转存或服务器代下
- 内置浏览器
- 站点内容搜索聚合
- DRM 内容破解
- 所有 `yt-dlp` 参数的完整可视化映射
- 真正意义上的“通用暂停/恢复”承诺

说明：首版可支持“取消、重试、重新下载”，是否提供严格意义的断点续传或暂停恢复，取决于 `yt-dlp` 与目标站点支持情况，不作为 MVP 强承诺能力。

## 7. 核心使用场景

## 7.1 场景一：快速下载单个视频

用户粘贴视频链接，应用自动解析标题、封面、时长和可用格式，用户选择“最佳画质”或指定分辨率，点击下载，任务进入队列并显示实时进度。

## 7.2 场景二：提取音频

用户粘贴链接，切换到“仅音频”，选择 `mp3`、`m4a` 或其他支持格式，设置音质后开始下载。

## 7.3 场景三：下载播放列表

用户粘贴播放列表链接，应用展示条目数量、总时长估算和前几个条目，允许用户选择全部下载或部分下载。

## 7.4 场景四：需要登录或 Cookie 的内容下载

用户在高级选项中选择“从浏览器读取 Cookie”或导入 Cookie 文件，应用完成解析并执行下载。

## 7.5 场景五：批量下载与失败恢复

用户一次性粘贴多个链接，应用建立任务队列；失败任务支持查看日志、识别常见原因并一键重试。

## 8. 功能需求

## 8.1 首页与输入区

功能要求：

- 提供明显的链接输入框
- 支持粘贴单个或多个 URL
- 支持识别单视频、播放列表、频道/合集等常见链接类型
- 支持“粘贴并解析”
- 支持从剪贴板快速导入

交互要求：

- 粘贴后自动进行格式校验
- 非法链接需即时提示
- 解析中需要有加载状态

## 8.2 媒体信息解析

功能要求：

- 基于 `yt-dlp` 获取媒体元信息
- 展示标题、封面、时长、上传者、站点、发布日期
- 对播放列表展示条目数与条目预览
- 支持列出可下载格式、字幕、封面信息

交互要求：

- 解析失败需给出明确错误
- 对“目标站点支持但当前链接无法解析”的情况，需要提示可能的 Cookie、地区限制、版本过旧等原因

## 8.3 下载模式选择

至少支持以下模式：

- 视频下载
- 仅音频
- 字幕下载
- 视频 + 字幕

默认策略：

- 默认提供“推荐模式”
- 普通用户默认不需要理解复杂格式表达式

## 8.4 格式与质量选择

功能要求：

- 支持“最佳质量”“平衡质量”“最低体积”等快捷选项
- 支持选择分辨率，如 `2160p`、`1440p`、`1080p`、`720p`
- 支持选择容器格式，如 `mp4`、`mkv`、`webm`
- 音频模式支持选择格式，如 `mp3`、`m4a`、`opus`、`flac`
- 在高级模式下，允许查看更完整的格式列表

交互要求：

- 普通模式以人能理解的方式展示，不直接暴露复杂 `-f` 表达式
- 高级模式可展示“编码、分辨率、码率、是否含音轨、文件大小估计”

## 8.5 字幕功能

功能要求：

- 支持下载字幕
- 支持下载自动生成字幕
- 支持字幕语言选择
- 支持字幕格式选择，如 `srt`、`ass`
- 支持“无字幕时继续下载视频”

## 8.6 封面与元数据

功能要求：

- 支持下载封面
- 支持将封面嵌入音频/视频文件
- 支持写入基础元数据

说明：

- 是否能稳定嵌入取决于输出格式与 `ffmpeg`/相关依赖支持，界面需有提示

## 8.7 播放列表与批量任务

功能要求：

- 支持播放列表整体下载
- 支持仅下载前 N 项
- 支持勾选部分条目下载
- 支持多链接批量导入
- 支持任务排队执行
- 支持并发数配置

交互要求：

- 播放列表解析后，用户应能清楚看到“总条目数”和“将实际下载多少项”

## 8.8 任务中心

功能要求：

- 展示等待中、下载中、已完成、失败、已取消任务
- 展示每个任务的标题、进度、速度、剩余时间、保存路径
- 支持取消任务
- 支持失败重试
- 支持打开文件所在目录
- 支持查看任务日志

说明：

- 首版不承诺跨站点、跨协议的一致暂停/恢复体验
- 若底层可恢复，应以“继续下载”形式提供，不作为强一致功能描述

## 8.9 保存与命名规则

功能要求：

- 支持选择默认保存目录
- 支持任务级保存目录覆盖
- 支持基础命名模板
- 支持按站点/作者/播放列表分类保存

首版建议提供的命名模板：

- 标题
- 标题 + ID
- 上传者/标题
- 播放列表/序号/标题

## 8.10 Cookie 与登录能力

功能要求：

- 支持加载 Netscape 格式 Cookie 文件
- 支持从浏览器读取 Cookie
- 至少在 UI 中支持常见浏览器选项：
  - Chrome
  - Chromium
  - Edge
  - Firefox
  - Safari
- 支持提示 Cookie 可能带来的隐私风险

交互要求：

- Cookie 相关配置必须明确告知保存方式
- 敏感信息默认不明文展示

## 8.11 环境检测与依赖管理

功能要求：

- 启动时检测 `yt-dlp` 是否可用
- 检测 `ffmpeg` / `ffprobe` 是否可用
- 检测 `yt-dlp-ejs` 与外部 JavaScript runtime 是否可用
- 检测版本是否过旧
- 缺失依赖时给出明确安装提示

建议能力：

- 提供“自动检测路径”
- 提供“手动指定二进制路径”
- 对 YouTube 下载场景，明确提示是否已满足完整支持条件
- 支持检测常见 runtime：
  - Deno
  - Node.js
  - Bun
  - QuickJS
- 后续版本可加入“自动更新 yt-dlp”

## 8.12 设置中心

至少包含：

- 默认保存目录
- 默认下载模式
- 默认格式策略
- 是否自动下载字幕
- 是否自动写入封面/元数据
- 并发数
- 代理设置
- 是否启用高级模式
- `yt-dlp` 路径
- `ffmpeg` 路径

## 8.13 日志与错误提示

功能要求：

- 保存任务级运行日志
- 对常见错误给出可读提示
- 支持复制完整日志

常见错误类型示例：

- `yt-dlp` 不存在
- `ffmpeg` 不存在
- 站点要求 Cookie/登录
- 网络错误
- 地区限制
- 目标格式不可合并
- 输出目录无权限
- 当前 `yt-dlp` 版本过旧

## 9. 非功能需求

## 9.1 易用性

- 新用户无需阅读文档即可完成首次下载
- 默认界面不应出现大量底层参数
- 关键路径尽量控制在 3 步内：
  - 粘贴链接
  - 选择模式/质量
  - 开始下载

## 9.2 性能

- 单链接解析响应应足够快
- 大播放列表解析时应有渐进式反馈
- 多任务下载时 UI 不应卡死

## 9.3 稳定性

- 单个任务失败不应拖垮整个应用
- 任务中断后历史记录仍可恢复查看
- 应用重启后可保留最近任务记录和用户设置

## 9.4 安全与隐私

- 不上传用户链接、Cookie、下载历史到远端
- 本地敏感配置应尽量安全保存
- 日志中避免直接暴露敏感凭证

## 9.5 可维护性

- UI 层与下载执行层分离
- 下载参数应通过统一配置模型生成
- 便于未来新增站点相关高级配置

## 10. 版本规划

## 10.1 V1 MVP

目标：先做一个用户真的能用起来的版本。

包含：

- 单链接下载
- 播放列表下载
- 批量导入
- 视频/音频/字幕模式
- 质量与格式选择
- 任务中心
- 基础日志
- 保存目录与命名模板
- Cookie 文件 / 浏览器 Cookie
- 依赖检测

## 10.2 V1.1

包含：

- 更好的播放列表选择器
- 更完整的历史记录
- 失败原因分类
- 高级格式列表
- 自动检查 `yt-dlp` 更新

## 10.3 V1.2+

可选扩展：

- 下载归档管理
- 任务预设
- 代理配置增强
- SponsorBlock 相关选项
- 频道/合集订阅式下载
- 更强的高级命令映射

## 11. 明确不做的设计误区

- 不把所有 CLI 参数直接原样平铺到界面
- 不把高级用户需求当作默认入口
- 不把“支持很多站点”误做成“每个站点都单独写页面”
- 不对受站点策略影响的能力做过度承诺
- 不把日志隐藏到用户找不到的地方

## 12. 建议的信息架构

推荐主界面结构：

- 顶部：链接输入区 + 快速下载操作
- 中部左侧：下载配置区
- 中部右侧：解析结果预览
- 底部或独立标签页：任务中心
- 设置页：依赖、目录、默认参数、Cookie、高级选项

推荐一级导航：

- 下载
- 任务
- 历史
- 设置

## 13. MVP 验收标准

满足以下条件可视为 MVP 达标：

- 用户首次打开应用，可以明确看到“输入链接并下载”的入口
- 用户可以下载单个公开视频到指定目录
- 用户可以切换到“仅音频”并导出 `mp3` 或 `m4a`
- 用户可以下载带字幕的视频，并选择字幕语言
- 用户可以下载播放列表，并看到条目数量
- 用户可以查看每个任务的进度、状态和日志
- 用户在缺少 `ffmpeg` 时，能收到明确提示
- 用户在需要 Cookie 的场景下，能找到导入入口
- 用户失败后可以一键重试任务

## 14. 风险与约束

- `yt-dlp` 的可用性依赖目标站点策略变化
- 部分站点需要额外 Cookie、浏览器指纹或更高版本依赖
- 音视频合并、封面嵌入、转码强依赖 `ffmpeg`
- 某些高级能力在不同平台上的表现可能不完全一致
- 浏览器 Cookie 读取在不同系统权限模型下可能存在差异
